How It Works
Drop a config. Point at a directory.
Events start streaming.

Define your file format in JSON. Conduo watches, validates every row, transforms, and streams to Kafka and JSONL. Fifteen minutes from clone to streaming.

1

Define Your Schema

One JSON file per data type. Map columns, set types, add validation rules, define derived calculations with evalexpr. Takes 10 minutes, not 10 days.

data_config/supplier_orders.json { "schema": "supplier_orders", "delimiter": ",", "fields": [ {"source": "supplier_id", "required": true}, {"source": "order_date", "type": "date"}, {"source": "quantity", "type": "integer"}, {"source": "unit_price", "type": "decimal"} ], "derived": [ {"field": "line_total", "expr": "quantity * unit_price"}, {"field": "vat", "expr": "line_total * 0.2"} ] }
2

Ingest

Watch a directory for incoming files — CSV, TSV, pipe-delimited, any separator. Or POST data to the REST API with API key auth. Both validate against the same schema.

File watcher: /data/incoming/*.csv → validate → stream Any delimiter · auto-detected per schema REST API (:8090): POST /api/ingest/supplier_orders Header: X-API-Key: {key} OpenAPI spec + Swagger UI Both paths: Same schema validation Same output sinks Same dashboard visibility
3

Stream & Monitor

Events stream to Kafka and JSONL simultaneously. Heartbeats every ~30 seconds to the controller. Dashboard shows throughput, recent files, rejected rows, and reasons.

Output (simultaneous): → kafka://conduo_events → /output/supplier_orders_20260226.jsonl Controller: Heartbeat every ~30s ● up / ● stale / ● down Fleet view across all instances Dashboard: Throughput · Recent files · Rejections

FAQ
Common questions

Everything you need to know about replacing your cron job.

What file formats are supported?

CSV, TSV, pipe-delimited, or any custom delimiter — configurable per schema. The file watcher detects new files automatically. You can also POST data via the REST API for programmatic ingestion.

What happens to bad data?

Bad rows are rejected with specific reasons — missing required field, wrong type, out of range. Valid rows continue through the pipeline. Nothing is silently dropped. The dashboard shows exactly what was rejected and why.

Is it really open source?

Yes. The entire engine — file watcher, REST API, validation, transforms, Kafka + JSONL sinks, local dashboard — is Apache 2.0 on GitHub. No feature gating, no time limits.

Why Rust?

Zero garbage collection means predictable latency. Memory safety without runtime overhead. Single static binary with no dependencies. Built by a maintainer of kafka-rust.

Can I use it without Kafka?

Yes. JSONL output is always available. If you just need validated, transformed files with monitoring — skip Kafka entirely. The dashboard and controller work either way.

How long does setup take?

Clone the repo, write a schema config for your file format, run start_platform.sh. Fifteen minutes from zero to streaming events. The config is the only thing you need to write.

What happens when the server restarts?

Self-healing recovery. Conduo tracks which files have been processed via checkpoints. On restart, it resumes from where it stopped — no duplicates, no missed files, no manual intervention.
