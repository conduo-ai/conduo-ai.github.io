How It Works
Configure. Ingest. Stream.

Define a JSON config per data type. Conduo handles parsing, validation, transformation, and delivery ‚Äî to Kafka, Event Hubs, or file output.

1

Configure Your Schema

Define field mappings, validation rules, and derived fields in a JSON config. One config per data type ‚Äî POS, products, inventory, stores, shipments.

{ "schema": "pos_transaction_line", "delimiter": "|", "fields": [ {"source": "STORE_ID", "target": "store_id", "required": true}, {"source": "ITEM_CODE", "target": "sku"}, {"source": "QTY", "target": "quantity", "type": "int"} ], "derived": [ {"field": "line_total", "formula": "qty * unit_price"} ] }
2

Ingest Data

Two ingestion modes: watch a directory for new files (pipe-delimited, CSV) or receive data via REST API endpoints. Conduo parses, validates, and normalises into consistent events.

File-based: Watch /data/pos/ for *.csv Auto-detect delimiter Parse ‚Üí validate ‚Üí transform API-based: POST /api/ingest/pos_transaction Content-Type: application/json ‚Üí validated, enriched, streamed
3

Stream Everywhere

Events publish to Kafka-compatible brokers ‚Äî local Redpanda, Azure Event Hubs, or the Event Hubs Emulator. Plus JSONL file output. Deterministic keys ensure idempotent delivery.

Outputs: ‚Üí kafka://broker/pos-events ‚Üí eventhubs://namespace/pos ‚Üí /output/pos_20260218.jsonl Every event includes: event_type, source_file, ingested_at, deterministic key

FAQ
Common questions

Everything you need to know about getting started.

What file formats are supported?

Pipe-delimited and CSV files. The file watcher monitors a directory and auto-processes new files as they arrive. Each file is parsed against its schema config for field mapping and validation.

What streaming brokers are supported?

Any Kafka-protocol compatible broker ‚Äî Redpanda for local/dev, Azure Event Hubs for production, plus the Event Hubs Emulator for testing. JSONL file output is always available as well.

Can I define custom data types?

Yes. Every schema is a JSON config ‚Äî field mappings, types, required flags, and derived field formulas. Create a new config file and Conduo handles the rest.

How does the dashboard work?

Svelte-powered real-time UI showing throughput (events/sec), time-series charts, and recent events by type with filter and expand. Full Metrics API and Swagger UI included.

How long does setup take?

Connect tier with one schema is live in hours ‚Äî define your config, point at a directory or API, and start streaming. Multi-schema deployments with Kafka typically take 4‚Äì6 weeks.
