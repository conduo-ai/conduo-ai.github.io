Integrations Controller
1

Sign Up & Configure

Create an account via the signup app. Define schemas in data_config/ — field mappings, types, validation rules, derived calculations via evalexpr. One JSON file per data type.

data_config/POS.json { "schema": "POS", "delimiter": "|", "fields": [ {"source": "STORE", "target": "store_id", "required": true}, {"source": "QTY", "type": "integer"}, {"source": "NET_AMT", "type": "decimal"} ], "derived": [ {"field": "gross", "expr": "net_amount * 1.2"} ] }
2

Ingest

Watch a directory for pipe-delimited files or POST data to the ingestion API at :8090 with API key auth. Both paths validate against the same schema configs.

File watcher: Data/*.txt → parse → validate → publish REST API (:8090): POST /api/ingest/POS Header: X-API-Key: {key} OpenAPI + Swagger UI Client tool: api_poster_rs --dir Data/ --api-key {key}
3

Stream & Monitor

Events publish to Kafka and JSONL simultaneously. The integration layer registers with the controller and sends heartbeats every ~30 seconds. Fleet dashboard shows every deployment.

Sinks (simultaneous): → kafka://conduo_events → /output/POS_20260218.jsonl Controller: POST /api/register → deployment ID POST /api/heartbeat (every ~30s) GET /api/deployments → fleet view Dashboard: ● up / ● stale / ● down

Platform
Built for operational data
at production scale

Rust-powered integration engine. Zero garbage collection. Config-driven. Dual sinks. Fleet monitoring with heartbeats. Everything you need to move data reliably.

🦀

Written in Rust

Zero-copy parsing. No garbage collector. Memory-safe without runtime overhead. Thousands of events per second with sub-millisecond latency. Built by a maintainer of kafka-rust.

📂

File Watcher

Watches directories for pipe-delimited files. Auto-detected and processed against schema configs. Drop a file, get events. One-shot or continuous mode.

🔌

REST Ingestion API

POST /api/ingest/{schema_name} with API key auth. OpenAPI spec and Swagger UI at :8090. Programmatic ingestion with validation and streaming in one call.

⚙️

Config-Driven Schemas

JSON configs in data_config/ define field mappings, types, required fields, and derived calculations via evalexpr. Add a new data type by adding a config file.

⚡

Kafka + JSONL Sinks

Publish to Redpanda or any Kafka-protocol broker on topic conduo_events. JSONL file output always available. Both sinks run simultaneously.

🛰️

Fleet Controller

Deployments register on startup and send heartbeats every ~30 seconds. Controller tracks status, licenses, customers. Svelte dashboard with deployment and customer views.

FAQ
Common questions

Everything you need to know about getting started.

What data sources are supported?

File watcher for pipe-delimited files and REST API with API key auth for programmatic ingestion — both live today. SAP OData and Dynamics 365 pollers are on the roadmap for the Scale tier.

Is it really open source?

Yes. The entire integration layer — Rust engine, file watcher, REST API, Kafka producer, JSONL sink, schemas, local dashboard, consumer tools — is Apache 2.0 on GitHub. No feature gating.

Why Rust?

Zero garbage collection means predictable latency. Memory safety without runtime overhead. A single static binary with no dependencies. Built by a maintainer of kafka-rust for production reliability.

Can I self-host everything?

The open source engine runs anywhere — your server, your cloud, your Docker host. The controller is a paid SaaS service. You can run the engine without the controller, you just won't have fleet monitoring or AI features.

How does the AI work?

AI features run in the controller, not the local engine. Schema inference analyses uploaded files server-side. Anomaly detection learns from event patterns across your tenant. All AI features are coming soon — join the waitlist.

Who's behind Conduo?

Built by Resolutis — 25+ years in enterprise integration, embedded systems, and Linux kernel development. Background includes Apple, Google, JLR, and NXP. Maintainer of kafka-rust.
