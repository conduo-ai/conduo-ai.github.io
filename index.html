How It Works
Deploy. Connect. Monitor.

Deploy an integration layer wherever your data lives. It ingests, transforms, and streams ‚Äî then reports back to the Conduo Controller so you always know what's running.

1

Configure & Deploy

Define schemas in JSON configs ‚Äî field mappings, validation, derived fields. Deploy the integration layer on-prem, in the cloud, or at the edge. Set a license key and controller URL.

{ "schema": "planning_application", "delimiter": ",", "fields": [ {"source": "REF", "target": "ref", "required": true}, {"source": "ADDRESS", "target": "address"}, {"source": "STATUS", "target": "status"} ], "derived": [ {"field": "decision_days", "formula": "decided - submitted"} ] }
2

Ingest & Stream

Watch directories for new files or receive data via REST API. Events are validated, transformed, and published to Kafka, Event Hubs, or JSONL. Local dashboard shows throughput in real-time.

File watcher: Watch /data/incoming/ for *.csv Parse ‚Üí validate ‚Üí transform ‚Üí stream REST API: POST /api/ingest/planning_application ‚Üí validated, enriched, streamed Outputs: ‚Üí kafka://broker/planning-events ‚Üí /output/planning_202602.jsonl
3

Monitor Remotely

Each integration layer registers with the Conduo Controller on startup and sends heartbeats every ~30 seconds. The controller tracks status, license validity, events/sec, and total events across all deployments.

Outbound-only model: Layer ‚Üí Controller (registration) Layer ‚Üí Controller (heartbeat ~30s) Controller tracks: ‚óè up / ‚óè stale / ‚óè down License validation Events/sec per deployment Total events across fleet

FAQ
Common questions

Everything you need to know about getting started.

What file formats are supported?

Pipe-delimited and CSV files. The file watcher monitors a directory and auto-processes new files as they arrive. Each file is parsed against its schema config for field mapping and validation.

What streaming brokers are supported?

Any Kafka-protocol compatible broker ‚Äî Redpanda for local/dev, Azure Event Hubs for production, plus the Event Hubs Emulator for testing. JSONL file output is always available as well.

Can I define custom data types?

Yes. Every schema is a JSON config ‚Äî field mappings, types, required flags, and derived field formulas. Create a new config file and Conduo handles the rest.

How does the dashboard work?

Svelte-powered real-time UI showing throughput (events/sec), time-series charts, and recent events by type with filter and expand. Full Metrics API and Swagger UI included.

How long does setup take?

Connect tier with one schema is live in hours ‚Äî define your config, point at a directory or API, and start streaming. Multi-schema deployments with Kafka typically take 4‚Äì6 weeks.
